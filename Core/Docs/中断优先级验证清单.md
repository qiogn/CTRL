# 中断优先级配置验证清单

## 验证目标
确认STM32F103C8T6步进电机控制项目的中断优先级配置正确，避免优先级冲突。

## 硬件信息
- MCU: STM32F103C8T6 (Cortex-M3)
- 中断优先级: 4位 (16级)
- 优先级分组: Group 0 (4位抢占优先级，0位子优先级)

## 配置验证

### 1. UART2中断配置 ✓
- **文件**: `Core/Src/usart.c`
- **行号**: 48-49
- **配置**: 
  ```c
  HAL_NVIC_SetPriority(USART2_IRQn, 0, 0);  // 抢占优先级0，子优先级0
  HAL_NVIC_EnableIRQ(USART2_IRQn);
  ```
- **状态**: 已配置，优先级最高

### 2. TIM3中断配置 ✓
- **文件**: `Core/Src/stepper_timer.c`
- **行号**: 75-76, 150
- **配置**:
  ```c
  // 初始化时配置
  HAL_NVIC_SetPriority(TIM3_IRQn, 6, 0);  // 抢占优先级6，子优先级0
  HAL_NVIC_EnableIRQ(TIM3_IRQn);
  
  // 反初始化时禁用
  HAL_NVIC_DisableIRQ(TIM3_IRQn);
  ```
- **状态**: 已配置，优先级中等

### 3. TIM2中断配置 ✓
- **文件**: `Core/Src/dual_stepper.c`
- **行号**: 89
- **配置**:
  ```c
  HAL_NVIC_SetPriority(TIM2_IRQn, 7, 0);  // 抢占优先级7，子优先级0
  // 注意：不启用中断，仅配置优先级
  ```
- **状态**: 已配置，优先级最低

## 优先级关系验证

### 预期优先级顺序
```
UART2(0) > TIM3(6) > TIM2(7)
```

## 已知限制

### 1. TIM2中断
- TIM2中断处理函数为空实现
- 中断可能被启用但无实际处理
- 建议：如果不需要TIM2中断，应在NVIC中禁用

### 2. 中断处理时间
- UART2中断应保持简短
- TIM3中断处理时间需监控
- 避免长时间中断阻塞

### 3. 优先级扩展
- 当前使用优先级0,6,7
- 剩余优先级可用于未来扩展
- 建议保留优先级1-5给高实时性任务
