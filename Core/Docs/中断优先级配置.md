# STM32F103C8T6 中断优先级配置

## 硬件信息
- **MCU**: STM32F103C8T6 (Cortex-M3)
- **中断优先级位**: 4位（16个优先级级别）
- **优先级数值**: 0-15，数值越小优先级越高
- **抢占优先级和子优先级**: 使用分组0（4位抢占优先级，0位子优先级）

## 中断优先级策略

### 设计原则
1. **实时性要求高的中断设高优先级**（UART接收数据）
2. **周期性、可延迟的中断设低优先级**（定时器PWM生成）
3. **避免优先级反转**：高优先级任务不应被低优先级任务阻塞
4. **系统异常保持最高优先级**（硬件固定）

### 具体配置

| 中断源 | 抢占优先级 | 子优先级 | 说明 |
|--------|------------|----------|------|
| **系统异常** | -1 (硬件固定) | - | HardFault, NMI等系统异常 |
| **UART2** | 0 | 0 | 视觉数据接收，最高优先级 |
| **TIM3** | 6 | 0 | 步进电机PWM生成，中等优先级 |
| **TIM2** | 7 | 0 | 备用定时器，最低优先级（如启用） |

### 优先级关系
```
UART2(0) > TIM3(6) > TIM2(7)
```

## 配置位置

### 1. UART2中断优先级
- **文件**: `Core/Src/usart.c`
- **函数**: `HAL_UART_MspInit`
- **配置**: 
  ```c
  HAL_NVIC_SetPriority(USART2_IRQn, 0, 0);
  HAL_NVIC_EnableIRQ(USART2_IRQn);
  ```

### 2. TIM3中断优先级
- **文件**: `Core/Src/stepper_timer.c`
- **函数**: `StepperTimer_Init`
- **配置**:
  ```c
  HAL_NVIC_SetPriority(TIM3_IRQn, 6, 0);
  HAL_NVIC_EnableIRQ(TIM3_IRQn);
  ```
- **禁用**: 在`StepperTimer_DeInit`中禁用

### 3. TIM2中断优先级
- **文件**: `Core/Src/dual_stepper.c`
- **函数**: `DualStepper_Init`
- **配置**:
  ```c
  HAL_NVIC_SetPriority(TIM2_IRQn, 7, 0);
  /* 注意：不启用中断，仅配置优先级 */
  ```

## 中断处理函数

### 1. UART2中断
- **文件**: `Core/Src/stm32f1xx_it.c`
- **函数**: `USART2_IRQHandler`
- **处理**: 调用`HAL_UART_IRQHandler`

### 2. TIM3中断
- **文件**: `Core/Src/stm32f1xx_it.c`
- **函数**: `TIM3_IRQHandler`
- **处理**: 调用`StepperTimer_TIM3_IRQHandler` → `HAL_TIM_IRQHandler`

### 3. TIM2中断
- **文件**: `Core/Src/stm32f1xx_it.c`
- **函数**: `TIM2_IRQHandler`
- **处理**: 调用`DualStepper_TIM2_IRQHandler`（空实现）

## 验证方法

### 1. 编译验证
```bash
# 检查编译是否成功
make build
```

### 2. 代码检查
- 确认所有中断优先级配置正确
- 确认中断使能/禁用配对正确
- 确认中断处理函数存在

### 3. 硬件测试
- **UART2测试**: 发送数据，确认接收正常
- **TIM3测试**: 启动PWM输出，确认电机运转
- **中断嵌套测试**: 同时触发UART和TIM3中断，确认UART优先处理

## 注意事项

### 1. 中断处理时间
- **UART2中断**: 应保持简短，只接收数据到缓冲区
- **TIM3中断**: 可能处理PWM更新，时间可控
- **避免长时间中断处理**

### 2. 共享资源保护
- UART接收缓冲区需要临界区保护
- 电机控制状态需要适当同步

### 3. 调试建议
- 使用调试器观察中断触发频率
- 监控中断处理时间
- 检查中断嵌套情况
